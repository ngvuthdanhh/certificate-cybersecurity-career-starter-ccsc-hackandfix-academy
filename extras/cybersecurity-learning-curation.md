# Extra 2 — Cybersecurity Learning Curation Guide

## 1. Purpose
Provide a curated learning improvement path focused on research, reasoning, validation, and responsible portfolio building.

---

## 2. Recommended Learning Focus Order
Foundations

Threat and incident reasoning

Hypothesis formulation

Academic reporting

Portfolio curation

Public knowledge contribution

Feedback integration

Continuous iteration

---

## 3. Content Sharing Principles
| Principle | Explanation |
|---|---|
| **Academic-first** | Prioritize clarity and correctness |
| **No exploit automation showcase** | Avoid enabling misuse |
| **Sector-aware risk** | Tailor insights to industry patterns |
| **Evidence-backed** | Always attach practical reasoning |
| **Portfolio integrity** | Organized and verifiable |
| **Feedback-driven** | Improve based on response |

---

## 4. Post-Learning Validation Output
After each learning milestone, produce:

✔ 1 hypothesis doc (60–100 words)
✔ 1 lab report (150–200 words)
✔ Confidence score (1–5)
✔ 1 portfolio index update line

---

## 5. Learning Quality Rating Rubric
| Score | Meaning |
|---|---|
| 5 | Lab-backed + strong correlation |
| 4 | Good reasoning + minor gaps |
| 3 | Basic understanding |
| 2 | Weak evidence or correlation |
| 1 | Must be relearned |
